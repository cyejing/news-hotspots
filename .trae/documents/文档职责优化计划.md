# 文档职责优化计划

## 问题分析

当前两个文件都包含"用户意图识别"内容，存在重复：

### 当前内容分布

**SKILL.md**（第 45-62 行）：
- 核心原则
- 意图判断规则表格（3 行）
- 判断优先级（3 条）

**execution-guide.md**（第 7-69 行）：
- 核心原则（重复）
- 意图判断流程
- 三种意图类型详细说明
- 判断优先级（重复）
- 典型场景示例表格（6 行）

### 存在的问题

1. **核心原则重复**：两个文件都写了相同的"不要主动展示已归档的 markdown"
2. **判断优先级重复**：两个文件都写了相同的优先级规则
3. **表格内容重叠**：意图判断表格和典型场景示例表格有重叠
4. **职责不清晰**：大模型不知道应该优先参考哪个文档

## 文档职责定位

### SKILL.md 职责

**定位**：主入口文档，快速参考

**应该包含**：
- 功能概览表格
- 核心判断规则（简洁）
- 引用详细文档的链接

**不应该包含**：
- 详细的触发条件列表
- 完整的流程说明
- 大量的示例

### execution-guide.md 职责

**定位**：详细执行指南，完整参考

**应该包含**：
- 完整的意图识别流程
- 详细的触发条件
- 判断优先级
- 典型场景示例
- 执行约束

**不应该包含**：
- 简化的判断规则（应该完整）

## 优化方案

### 方案：职责分离 + 明确引用

#### SKILL.md 优化

保留简洁的意图识别规则，但明确引用详细文档：

```markdown
## 用户意图识别

**核心原则**：不要主动展示已归档的 markdown 给用户，已看过的内容重复看意义不大。

### 快速判断规则

| 用户意图         | 执行操作                          |
|--------------|-------------------------------|
| 查看今日/本周热点   | 运行 `run-pipeline.py` 抓取新热点    |
| 当天换一批新闻    | 运行 `merge-hotspots.py` 生成新热点  |
| 查看归档（明确要求） | 读取归档 markdown 并翻译输出            |

**详细判断规则**：参见 `references/execution-guide.md` 的"用户意图识别"章节。
```

#### execution-guide.md 优化

保留完整的意图识别章节，不需要修改。

### 优化后的效果

1. **职责清晰**：
   - SKILL.md：快速参考，一目了然
   - execution-guide.md：详细指南，深入理解

2. **避免重复**：
   - 核心原则只在 SKILL.md 中出现
   - 判断优先级只在 execution-guide.md 中出现
   - 表格内容各有侧重

3. **引用明确**：
   - SKILL.md 明确引用 execution-guide.md
   - 大模型知道在哪里找到详细信息

## 实施步骤

1. 修改 SKILL.md 第 45-62 行，简化意图识别规则
2. 在 SKILL.md 中添加对 execution-guide.md 的引用
3. execution-guide.md 保持不变

## 预期结果

- SKILL.md：简洁明了，快速识别意图
- execution-guide.md：详细完整，深入理解流程
- 两个文件职责清晰，互为补充，不重复
